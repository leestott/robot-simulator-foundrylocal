Building a Voice-Controlled Robot Simulator with On-Device AI

A practical guide for AI engineers and developers using Foundry Local, the Microsoft Agent Framework, and PyBullet — no cloud, no API keys.

Why This Matters

Most AI demos send every request to a cloud API. That means latency, costs, and data leaving your machine. Foundry Local changes the equation: you get an OpenAI-compatible endpoint running entirely on-device. Pair it with a multi-agent framework and a physics simulator, and you have a complete voice-to-action pipeline with zero cloud dependencies.

Setting Up Foundry Local

from foundry_local import FoundryLocalManager
import openai

manager = FoundryLocalManager("qwen2.5-coder-0.5b")

client = openai.OpenAI(
    base_url=manager.endpoint,
    api_key=manager.api_key,
)

resp = client.chat.completions.create(
    model=manager.get_model_info("qwen2.5-coder-0.5b").id,
    messages=[{"role": "user", "content": "pick up the cube"}],
    max_tokens=128,
    stream=True,
)

The SDK auto-selects the best hardware backend (CUDA GPU → QNN NPU → CPU). No configuration needed.

Voice Input Pipeline

Voice commands follow three stages:

  1. Browser capture — MediaRecorder captures audio, client-side resamples to 16 kHz mono WAV
  2. Server transcription — Foundry Local Whisper (ONNX, cached after first load) with automatic 30 s chunking
  3. Command execution — transcribed text goes through the same Planner → Safety → Executor pipeline

The mic button (🎤) only appears when a Whisper model is cached or loaded. Whisper models are filtered out of the LLM dropdown.

Getting Started

# 1. Install Foundry Local
winget install Microsoft.FoundryLocal    # Windows
brew install foundrylocal                # macOS

# 2. Download models
foundry model run qwen2.5-coder-0.5b    # Chat brain (~4 s inference)
foundry model run whisper-base           # Voice input (194 MB)

# 3. Clone & setup
git clone https://github.com/leestott/robot-simulator-foundrylocal
cd robot-simulator-foundrylocal
.\setup.ps1                              # or ./setup.sh on macOS/Linux

# 4. Run
python -m src.app --web --no-gui         # → http://localhost:8080

Key Takeaways

  1. On-device AI is production-ready. Foundry Local serves models via an OpenAI-compatible API. Swap base_url and you're done.
  2. Multi-agent beats monolithic. Four focused agents with JSON schema contracts are more reliable than one prompt.
  3. Voice is just another input. Same pipeline for text and speech — only the transcription step differs.
  4. Model size drives latency. A 0.5B model at ~4 s is 10× faster than a 7B model at ~45 s.
  5. Async-first for real-time. asyncio.create_task + WebSocket keeps the UI responsive.
